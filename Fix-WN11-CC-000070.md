# WN11-CC-000070 - Turn On Virtualization-Based Security

## Overview

This security setting requires Windows to enable **Virtualization-Based Security (VBS)** and configure the platform security level to use **Secure Boot** or **Secure Boot and DMA Protection**.

VBS leverages hardware virtualization features to create an isolated memory region that protects critical security components, including Credential Guard.

> **Note:** Virtual Desktop Infrastructure (VDI) environments that are non-persistent and refreshed or deleted upon logoff may be considered Not Applicable (NA) for this requirement.

---

## STIG Requirement

| Setting                               | Required Value                                |
| ------------------------------------- | --------------------------------------------- |
| Turn On Virtualization-Based Security | Enabled                                       |
| Select Platform Security Level        | Secure Boot or Secure Boot and DMA Protection |

---

## Prerequisites

Verify the system supports:

* UEFI firmware
* Secure Boot enabled
* Trusted Platform Module (TPM)
* Hardware virtualization enabled in BIOS/UEFI
* Hyper-V virtualization support

---

## Manual Remediation

### Configure via Local Group Policy

1. Open the Local Group Policy Editor:

   * Press **Win + R**
   * Type:

   ```text
   gpedit.msc
   ```

   * Select **OK**

2. Navigate to:

   ```text
   Computer Configuration
   └── Administrative Templates
       └── System
           └── Device Guard
   ```

3. Open:

   ```text
   Turn On Virtualization-Based Security
   ```

4. Select:

   ```text
   Enabled
   ```

5. Under **Select Platform Security Level**, choose one of the following:

   ```text
   Secure Boot
   ```

   or

   ```text
   Secure Boot and DMA Protection
   ```

6. Select **Apply**.

7. Select **OK**.

---

## Enterprise Remediation (Group Policy)

1. Open **Group Policy Management Console (GPMC)**.

2. Edit an existing Group Policy Object or create a new one.

3. Navigate to:

   ```text
   Computer Configuration
   └── Policies
       └── Administrative Templates
           └── System
               └── Device Guard
   ```

4. Configure:

   ```text
   Turn On Virtualization-Based Security = Enabled
   ```

5. Configure:

   ```text
   Select Platform Security Level =
   Secure Boot
   ```

   or

   ```text
   Secure Boot and DMA Protection
   ```

6. Link the GPO to the appropriate Organizational Unit (OU).

7. Force Group Policy refresh:

   ```cmd
   gpupdate /force
   ```

8. Reboot the system if required.

## Verification

### Verify Using System Information

1. Open:

   ```text
   msinfo32.exe
   ```

2. Verify the following values:

   ```text
   Virtualization-based security: Running
   ```

   and

   ```text
   Secure Boot State: On
   ```

---

### Verify Using PowerShell

Run:

```powershell
Get-CimInstance -ClassName Win32_DeviceGuard
```

Review the output to confirm:

```text
VirtualizationBasedSecurityStatus = 2
```

A value of **2** indicates VBS is running.

---

## Compliance Validation

The system is compliant when:

* Turn On Virtualization-Based Security is configured as **Enabled**
* Platform Security Level is configured as:

  * **Secure Boot**, or
  * **Secure Boot and DMA Protection**
* Virtualization-Based Security is operational

---

## Special Considerations

### Non-Persistent VDI

This requirement may be marked **Not Applicable (NA)** when:

* The virtual desktop is deleted at logoff, or
* The virtual desktop is refreshed to a clean state at logoff

and the environment cannot support the hardware requirements necessary for Credential Guard and VBS.
