# WN11-CC-000068 - Remote Host Allows Delegation of Non-Exportable Credentials

## Overview

This security setting ensures that Remote Credential Guard and related credential delegation mechanisms use **non-exportable credentials**, reducing the risk of credential theft during remote authentication sessions.

## Vulnerability Discussion

Enabling this policy helps protect user credentials by ensuring delegated credentials cannot be exported from the remote system. This provides additional protection against credential theft and unauthorized reuse.

## Manual Remediation

### Configure via Local Group Policy

1. Open the **Local Group Policy Editor**:
   - Press **Win + R**
   - Type `gpedit.msc`
   - Select **OK**

2. Navigate to:

   ```text
   Computer Configuration
   └── Administrative Templates
       └── System
           └── Credentials Delegation
   ```

3. Double-click:

   ```text
   Remote host allows delegation of non-exportable credentials
   ```

4. Select:

   ```text
   Enabled
   ```

5. Select **Apply**.

6. Select **OK**.

7. Update Group Policy:

   ```cmd
   gpupdate /force
   ```

---


The system is compliant when **Remote host allows delegation of non-exportable credentials** is configured to **Enabled**.
