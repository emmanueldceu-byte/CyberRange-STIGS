# WN11-CC-000090 - Group Policy Objects Must Be Reprocessed Even If They Have Not Changed

## Overview

This STIG requires Windows to reapply Group Policy settings during background refresh even when the Group Policy Objects (GPOs) have not changed.

**Required Setting:** Enabled

---

# Manual Remediation

## Step 1: Open Local Group Policy Editor

Open the Windows Search bar and type:

```text
gpedit.msc
```

Press **Enter**.

---

## Step 2: Configure Group Policy Processing

Navigate to:

```text
Computer Configuration
└─ Administrative Templates
   └─ System
      └─ Group Policy
         └─ Configure  registry policy  processing
```

Open the policy and configure:

```text
Enabled
```

Then select:

```text
Process even if the Group Policy objects have not changed
```

Click:

```text
Apply
OK
```

---

## Step 3: Update Group Policy

Open Command Prompt as Administrator and run:

```cmd
gpupdate /force
```

---

# PowerShell Auto Remediation

Save as:

```text
Fix-WN11-CC-000090.ps1
```

```powershell
<#
STIG: WN11-CC-000090
Configure registry Policy Processing
Process even if GPOs have not changed
#>

Write-Host "Applying WN11-CC-000090 remediation..." -ForegroundColor Green

$RegPath = "HKLM:\SOFTWARE\Policies\Microsoft\Windows\System"

if (!(Test-Path $RegPath)) {
    New-Item -Path $RegPath -Force | Out-Null
}

# Configure registry policy processing
New-ItemProperty `
    -Path $RegPath `
    -Name "ProcessEvenIfGPOsHaveNotChanged" `
    -PropertyType DWord `
    -Value 0 `
    -Force | Out-Null

gpupdate /force

Write-Host "Remediation completed." -ForegroundColor Green
```

---

# Validation

## Method 1 - Registry Validation

Run:

```powershell
Get-ItemProperty `
-Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows\System"
```

Expected result:

```text
ProcessEvenIfGPOsHaveNotChanged : 0
```

---

## Method 2 - GPResult

Run:

```cmd
gpresult /h C:\Temp\GPReport.html
```

Open the report and verify:

```text
Computer Configuration
→ Administrative Templates
→ System
→ Group Policy
→ Configure security policy processing

State: Enabled
Process even if the Group Policy objects have not changed: Enabled
```
