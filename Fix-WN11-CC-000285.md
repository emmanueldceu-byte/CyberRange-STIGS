# WN11-CC-000285 – Require Secure RPC Communication

## Overview

This control ensures that Remote Desktop Session Host (RDSH) enforces secure RPC communication, reducing the risk of insecure Remote Procedure Call (RPC) traffic being intercepted or manipulated.

---

## Compliance Requirement

### Policy Path
Computer Configuration
→ Administrative Templates
→ Windows Components
→ Remote Desktop Services
→ Remote Desktop Session Host
→ Security


### Setting

- **Require secure RPC communication**

---

## Manual Remediation Steps (Local System)

### Step 1: Open Group Policy Editor

1. Press `Win + R`
2. Type:gpedit.msc

3. Press **Enter**

---

### Step 2: Navigate to Policy Path

Go to:
Computer Configuration
→ Administrative Templates
→ Windows Components
→ Remote Desktop Services
→ Remote Desktop Session Host
→ Security


---

### Step 3: Configure Policy

1. Locate **Require secure RPC communication**
2. Double-click the policy
3. Select **Enabled**
4. Click **Apply**
5. Click **OK**

---

### Step 4: Apply Policy Update

Run the following command:

```bash
gpupdate /force
```

Rescan the VM to ensure it is fixed.
